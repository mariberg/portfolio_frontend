Tilawa — Quran Study Companion

Type
Hackathon Project
Role
Solo — Full Stack
AWS Bedrock Nova Lite Lambda DynamoDB CloudFormation Flutter AI Personalisation OAuth2 PKCE
The problem

Muslims reciting the Quran often have limited context about what they're about to read — especially those still developing their Arabic. Generic apps provide translations but nothing personalised to the reader's proficiency level or reading history. The result is a passive experience rather than an engaged one.

The approach

Tilawa generates a personalised AI briefing before each recitation session. Based on the user's Arabic level and their DynamoDB-stored reading history, Nova Lite produces a contextual summary of the upcoming passage and a set of vocabulary flashcards — filtered to words the user hasn't encountered before.

Key technical decisions
  • Cross-account STS for Bedrock — rather than direct IAM access, the Lambda assumes a role in a separate AWS account to call Bedrock. An enterprise-grade security pattern that keeps AI costs isolated and auditable.
  • Two-pass keyword filtering — vocabulary is filtered first by Arabic frequency level, then against the user's DynamoDB history. This produces genuinely novel flashcards per session rather than generic wordlists.
  • Single-table DynamoDB design — user sessions, reading history, and vocabulary data colocated in one table with PK/SK patterns. Requires upfront data modelling discipline but scales cleanly.
  • Property-based testing on both sides — fast-check (Node.js) and glados (Flutter) generate edge-case inputs automatically. Uncommon in hackathon projects; included because correctness of the filtering pipeline matters.
